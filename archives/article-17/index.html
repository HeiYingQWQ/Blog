ClawHub 完全指南：从安装到发布你的第一个 Skill

核心要点

  • ClawHub 是 OpenClaw 生态的官方 Skill 仓库，类似 npm 但专为 AI Agent 设计
  • 支持 Skill 的搜索、安装、更新、发布全流程管理
  • 开发 Skill 只需遵循简单的目录结构和 SKILL.md 规范
  • 发布到 ClawHub 后，其他 OpenClaw 用户可直接安装使用

什么是 ClawHub

ClawHub 是 OpenClaw 框架的官方 Skill 仓库，你可以把它理解为：

类比 说明
npm for Node.js ClawHub 为 OpenClaw 提供 Skill 包管理
PyPI for Python 开发者可以发布和共享 Skill
VS Code Marketplace 用户一键安装，即装即用

为什么需要 ClawHub

在没有 ClawHub 之前，OpenClaw 用户想要使用别人开发的 Skill，需要：

  1. 去 GitHub 搜索相关仓库
  2. 手动下载或 clone
  3. 按照 README 手动配置
  4. 解决各种依赖问题

有了 ClawHub 之后：

一行命令，自动完成搜索、下载、安装、配置。

发布到 ClawHub

发布前检查清单

  • SKILL.md 完整，包含描述、安装、使用、API 参考
  • package.json 字段完整，版本号正确
  • 所有依赖已声明
  • 本地测试通过
  • 没有敏感信息（密码、API Key 等）
  • 有合适的开源许可证

版本号规范

遵循语义化版本（SemVer）：主版本.次版本.修订号

版本变化 说明 示例
主版本 不兼容的 API 修改 1.0.0 → 2.0.0
次版本 向下兼容的功能新增 1.0.0 → 1.1.0
修订号 向下兼容的问题修复 1.0.0 → 1.0.1

发布命令

发布流程：

  1. 读取 package.json 验证元数据
  2. 打包 Skill 目录（排除 .git、node_modules 等）
  3. 上传到 ClawHub
  4. 自动运行基础质检（语法检查、安全扫描）
  5. 发布成功，获得公开 URL

最佳实践

1. 保持单一职责

每个 Skill 只做一件事，做好一件事。不要把十个功能塞到一个 Skill 里。

2. 详细的文档

假设用户第一次接触你的 Skill，提供：

  • 清晰的功能描述
  • 安装步骤
  • 使用示例（至少 3 个）
  • 常见问题 FAQ

常见问题

Q: Skill 和普通的 Python 脚本有什么区别？

A: 主要区别：

特性 普通脚本 Skill
安装方式 手动下载 openclaw skill install
依赖管理 手动处理 自动解析安装
入口标准 任意 统一的 scripts/main.py
文档规范 无要求 必须 SKILL.md
版本控制 自己维护 ClawHub 统一管理

Q: 发布 Skill 需要审核吗？

A: 基础审核是自动化的（安全检查、语法检查）。对于热门分类或高权限 Skill，可能需要人工审核。通常发布后几分钟内即可使用。

Q: 可以发布私有 Skill 吗？

A: 可以。openclaw skill publish --private 发布的 Skill 只有你和授权用户可见。适合公司内部使用。
