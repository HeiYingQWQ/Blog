上周五晚上，老板临下班前说”那个Vue项目能不能今晚上线？客户明天要演示”。我当时就慌了——本地跑得好好的，一上服务器就 500 错误，折腾到凌晨两点还没搞定，最后还是群里的大佬指了一条明路。今天就把这段血泪史整理出来，希望能帮到同样被部署搞得头秃的朋友 🐾

为什么生产环境总在关键时刻崩盘

刚开始我以为是自己代码写烂了，查了半天才发现是典型的Vue SPA 部署三件套问题：

  1. 路由模式搞错了：开发环境用 history 模式没问题，但生产环境没有服务器配置刷新就 404
  2. 跨域被坑了：本地 proxy 转发风生水起，一上 Nginx 就 CORS 报错
  3. 缓存没配置：改了代码用户还是看到旧版本，客户以为我没更新

好家伙，这不就是新手必踩的坑吗？

免费平替：开源 CDN 方案让首屏加载提升 300%

如果服务器带宽感人，静态资源可以丢到免费 CDN 上加速：

Cloudflare CDN（免费版够用）

  1. 注册 Cloudflare 账号，添加你的域名
  2. 把域名的 NS 记录指向 Cloudflare
  3. 在 DNS 设置里勾选 Proxied（小云朵图标）

搞定！所有流量都会经过 Cloudflare 的全球节点，国内访问会走就近节点（虽然不如阿里云 CDN 快，但免费嘛）。
