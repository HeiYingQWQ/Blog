你有没有想过，如何让你的 OpenClaw AI 小狗学习新技能？今天我就来教你写一个简单但实用的自定义技能——天气查询，只需要 30 分钟！

技能架构：SKILL.md + scripts 目录结构

首先，我们需要了解 OpenClaw 技能的基本架构。每个技能都有两个核心部分：

1. SKILL.md（技能文档）

这是技能的说明书，告诉 AI 如何使用这个技能。它包含：

  • 技能名称和描述
  • 适用场景
  • 使用方法
  • 注意事项

2. scripts 目录（技能脚本）

这是技能的实现部分，包含可执行的脚本文件。这些脚本会被 AI 调用，执行实际的任务。

调试与发布：技能测试和共享

1. 技能测试

在开发过程中，你需要不断测试你的技能，确保它能正常工作。你可以：

  • 直接调用脚本来测试
  • 在 OpenClaw 中通过聊天测试
  • 邀请朋友帮助测试

2. 技能发布

当你的技能测试通过后，你可以将其发布，让更多人使用。

共享给他人

你可以将你的技能文件夹打包分享给其他人，或者上传到 GitHub 仓库。

提交到 ClawHub（OpenClaw 官方技能仓库）

如果你希望你的技能被更多人发现，你可以提交到 ClawHub。

3. 版本管理

随着技能的不断改进，你需要管理版本。建议使用 Git 进行版本控制。

总结

写一个简单的 OpenClaw 自定义技能其实并不难！只需要：

  1. 了解技能架构（SKILL.md + scripts）
  2. 明确技能接口（如何调用）
  3. 实现脚本逻辑
  4. 测试和发布

今天的天气查询技能只是一个简单的示例。你可以根据这个思路，开发更多实用的技能，如翻译、计算、提醒等。

学习是一个持续的过程。 随着你对 OpenClaw 的理解加深，你可以写更复杂、功能更强大的技能。记住，技能开发是一个迭代的过程，不要害怕犯错误，不断尝试和改进！
