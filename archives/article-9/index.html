OpenClaw 配置文件完全解析

推荐理由：配置是 OpenClaw 的核心，很多用户卡在这一步

OpenClaw 用户，需要理解配置体系，一定遇到过这个痛点。今天手把手带你搞定「OpenClaw 配置」，全程实操。

前置条件

开始之前，请确认：

  • 已安装 OpenClaw 网关
  • 服务器环境已就绪
  • 具备基本的命令行操作能力

提示：本文基于最新稳定版本编写，所有配置均经过实测。

第一步：config.yml 结构总览

OpenClaw 的主配置文件是 config.yml，位于项目根目录。

基础结构

关键配置项

配置项 说明 示例
gateway.port 网关监听端口 8080
models.default 默认 AI 模型 openrouter
skills.path 技能目录路径 skills/blog-writer
cron.schedule Cron 表达式 "0 9 * * *"

第二步：Gateway 与 Models 配置详解

Gateway 配置

Gateway 是 OpenClaw 的核心服务，负责处理所有外部请求。

注意：

  • host: 0.0.0.0 表示监听所有网卡
  • 生产环境建议启用 TLS（enabled: true）
  • 端口冲突时修改 port 并放行防火墙

Models 配置

OpenClaw 支持多个 AI 模型提供商，通过 models 配置管理。

最佳实践：

  • API key 使用环境变量 ${VAR}，不要硬编码
  • site_url 和 site_name 用于模型提供商统计
  • 可以配置多个模型，default 指定默认使用的模型
