## 引言
随着技术发展，OpenClaw 已经成为开发者必备的 AI 助手框架。本文将手把手带你从零开始，完成 OpenClaw 网关的搭建与配置。全程基于最新稳定版本，适合新手跟随。

## 前置条件
在开始之前，请确保你拥有：

- [x] 一台运行 Linux/macOS/Windows WSL2 的机器（OpenClaw 目前不支持纯 Windows）
- [x] Node.js 18+ 和 Python 3.10+
- [x] 已安装 Git（建议配置 SSH 密钥）
- [x] 基本的命令行操作能力和权限管理概念
- [x] （可选）一个 OpenRouter 账号用于获取模型 API Key

## 第一步：安装 OpenClaw CLI

**方式一：通过 npm 全局安装（推荐）**

```bash
npm install -g @openclaw/cli
```

**方式二：从源码构建**

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
npm install
npm run build
npm link  # 或复制到 /usr/local/bin
```

验证安装：

```bash
openclaw --version
```

## 第二步：初始化网关工作区

OpenClaw 的所有配置和数据都存储在 `~/.openclaw/workspace`。运行初始化命令：

```bash
openclaw gateway init
```

这会生成：
- `gateway.yml`：网关主配置文件
- `workspace/`：工作区（存放日志、技能等）

## 第三步：配置 Models（模型 providers）

编辑 `~/.openclaw/workspace/gateway.yml`，添加模型配置：

```yaml
models:
  default: openrouter/anthropic/claude-3-opus  # 默认模型
  fallbacks:
    - openrouter/openai/gpt-4
    - openrouter/google/gemini-2.5-pro

providers:
  openrouter:
    api_key: ${OPENROUTER_API_KEY}  # 通过环境变量注入
```

**重要提示**：
- API Key 写在环境变量中，不要提交到 Git
- 模型标识符必须与提供商一致（可在 OpenRouter 控制台查看）
- 可使用 `openclaw models list` 测试连接性

## 第四步：配置 Gateway 通道（Channels）

根据你的使用场景，在 `gateway.yml` 的 `channels` 中启用对应通道。

**示例 1：仅使用 CLI（最简单）**

```yaml
channels:
  - cli
```

**示例 2：接入 Telegram**

```yaml
channels:
  - telegram

telegram:
  token: ${TELEGRAM_BOT_TOKEN}
  allowed_chats: []  # 空数组表示所有聊天
```

要获取 Telegram Bot Token：
1. 在 Telegram 中搜索 @BotFather
2. 发送 `/newbot` 并按提示创建
3. 将 Token 设置为环境变量 `TELEGRAM_BOT_TOKEN`

**示例 3：启用 Web 控制台**

```yaml
channels:
  - webchat

webchat:
  port: 8080
```

详细通道配置参考 [OpenClaw Channels 文档](https://docs.openclaw.ai/channels)。

## 第五步：启动网关服务

```bash
openclaw gateway start
```

默认监听：
- HTTP：3000（REST API）
- WebSocket：3001

查看状态：

```bash
openclaw gateway status
```

**调试**：初次启动建议加上 `--debug` 查看详细日志：

```bash
openclaw gateway start --debug
```

如果修改了 `gateway.yml`，重启或重载使配置生效：

```bash
openclaw gateway restart   # 重启
# 或
openclaw gateway reload     # 热重载（部分配置）
```

## 第六步：基本用法

**命令行交互（CLI）**：

```bash
# 发送消息给默认模型
openclaw message send --channel cli "你好，我是小白"

# 指定模型
openclaw message send --channel cli --model openrouter/anthropic/claude-3-opus "写一个 Hello World"
```

**Telegram / Discord**：
- 将 Bot 添加到群组或个人聊天
- 直接发送消息即可交互

**技能管理**：

```bash
# 查看已安装技能
openclaw skills list

# 从 ClawHub 安装新技能
openclaw skills install cli  # 示例

# 卸载技能
openclaw skills uninstall <skill-name>
```

**设备配对**（如果使用移动端 App）：

```bash
openclaw devices list       # 查看待批准请求
openclaw devices approve <request-id>   # 批准配对
```

## 常见问题

### 1. gateway 启动失败，提示端口占用
使用以下命令查看占用端口的进程：

```bash
lsof -i:3000   # macOS
netstat -tulpn | grep :3000  # Linux
```

解决方案：
- 停止占用端口的进程
- 或在 `gateway.yml` 中修改 `server.port` 为其他端口

### 2. 消息通道连接不上（Telegram/Discord）
检查点：
- `gateway.yml` 中的 token 是否正确
- 环境变量是否导出（如 `export TELEGRAM_BOT_TOKEN=xxx`）
- Bot 是否在对应聊天中有权限（\"Allow Groups\" 已开启）
- 网络是否能访问外部 API

### 3. 技能安装后不生效或加载失败
- 某些技能需要网关重启：`openclaw gateway restart`
- 或重载技能：`openclaw gateway reload`
- 查看日志 `~/.openclaw/logs/` 确认具体错误

### 4. API 调用返回 401/403
- 确认提供商 API Key 有效且有足够额度
- 某些模型（如 Claude 3 Opus）需要单独开通权限
- 检查 `gateway.yml` 的模型标识符拼写

### 5. 中文消息乱码
- 确保终端/聊天软件使用 UTF-8 编码
- 在 `gateway.yml` 中添加 `locale: zh-CN`
- 使用 `export LANG=zh_CN.UTF-8` 设置系统语言
