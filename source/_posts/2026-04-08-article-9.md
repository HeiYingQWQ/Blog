# OpenClaw 配置文件完全解析

> **推荐理由**：配置是 OpenClaw 的核心，很多用户卡在这一步

OpenClaw 用户，需要理解配置体系，一定遇到过这个痛点。今天手把手带你搞定「OpenClaw 配置」，全程实操。

## 前置条件

开始之前，请确认：

- [ ] 已安装 OpenClaw 网关
- [ ] 服务器环境已就绪
- [ ] 具备基本的命令行操作能力

> **提示**：本文基于最新稳定版本编写，所有配置均经过实测。

## 第一步：config.yml 结构总览

OpenClaw 的主配置文件是 `config.yml`，位于项目根目录。

### 基础结构

```yaml
gateway:
  port: 8080
  host: 0.0.0.0

models:
  default: openrouter
  openrouter:
    api_key: ${OPENROUTER_API_KEY}

skills:
  - path: skills/blog-writer
    enabled: true

cron:
  - name: daily-report
    schedule: "0 9 * * *"
    action:
      kind: systemEvent
      text: "生成日报"
```

### 关键配置项

| 配置项 | 说明 | 示例 |
|--------|------|------|
| `gateway.port` | 网关监听端口 | `8080` |
| `models.default` | 默认 AI 模型 | `openrouter` |
| `skills.path` | 技能目录路径 | `skills/blog-writer` |
| `cron.schedule` | Cron 表达式 | `"0 9 * * *"` |

## 第二步：Gateway 与 Models 配置详解

### Gateway 配置

Gateway 是 OpenClaw 的核心服务，负责处理所有外部请求。

```yaml
gateway:
  port: 8080
  host: 0.0.0.0
  tls:
    enabled: false
    cert_path: "/path/to/cert.pem"
    key_path: "/path/to/key.pem"
```

**注意**：
- `host: 0.0.0.0` 表示监听所有网卡
- 生产环境建议启用 TLS（`enabled: true`）
- 端口冲突时修改 `port` 并放行防火墙

### Models 配置

OpenClaw 支持多个 AI 模型提供商，通过 `models` 配置管理。

```yaml
models:
  default: openrouter
  openrouter:
    api_key: ${OPENROUTER_API_KEY}
    site_url: https://your-site.com
    site_name: "My OpenClaw"
```

**最佳实践**：
- API key 使用环境变量 `${VAR}`，不要硬编码
- `site_url` 和 `site_name` 用于模型提供商统计
- 可以配置多个模型，`default` 指定默认使用的模型

## 第三步：技能配置与权限管理

### 技能目录结构

每个技能是一个独立目录，必须包含：

```
skills/
└── my-skill/
    ├── SKILL.md          # 技能说明
    ├── scripts/          # 可执行脚本
    └── references/       # 辅助文件（可选）
```

### SKILL.md 必需字段

```yaml
---
name: my-skill
description: 技能描述
version: "1.0.0"
metadata:
  openclaw:
    emoji: "🛠️"
    requires:
      bins: ["python3"]
---
```

### 权限控制

OpenClaw 支持基于技能的文件系统权限。在 `config.yml` 中可以设置：

```yaml
security:
  skills:
    allowed_paths:
      - "/var/data"
      - "/tmp"
    deny_patterns:
      - "*.secret"
```

## 第四步：环境变量与密钥安全

### 使用环境变量

敏感信息（API keys、密码）必须通过环境变量注入：

```bash
export OPENROUTER_API_KEY="sk-xxxxx"
export DATABASE_URL="postgres://user:pass@localhost/db"
```

在 `config.yml` 中引用：

```yaml
models:
  openrouter:
    api_key: ${OPENROUTER_API_KEY}
```

### 多环境配置

开发/测试/生产环境使用不同的配置文件：

```bash
# 启动时指定配置文件
openclaw gateway --config config.dev.yml
openclaw gateway --config config.prod.yml
```

配置文件示例 `config.prod.yml`（继承基础配置）：

```yaml
# config.prod.yml
import: base.yml

gateway:
  port: 443
  tls:
    enabled: true
    cert_path: "/etc/ssl/certs/your-cert.pem"
    key_path: "/etc/ssl/private/your-key.pem"

models:
  openrouter:
    api_key: ${OPENROUTER_PROD_KEY}
```

## 第五步：常见问题与调试

### 配置文件语法错误

```bash
# 检查 YAML 语法
python3 -c "import yaml; yaml.safe_load(open('config.yml'))"
```

### 端口被占用

```bash
# 查看端口占用
lsof -i :8080
# 或修改 config.yml 使用其他端口
```

### 技能加载失败

检查日志：
```bash
journalctl -u openclaw -f
# 或查看技能日志
tail -f skills/my-skill/skill.log
```

### 重启服务

```bash
systemctl restart openclaw
# 或直接运行
openclaw gateway restart
```
