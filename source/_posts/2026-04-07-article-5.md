## 核心配置：cron + skills

OpenClaw 的自动化靠两块拼图：

1. **cron 任务**：定时触发（每 N 分钟/小时/天）
2. **skills**：具体执行动作（exec、message 等）

我在 `openclaw.json` 里设置：

```json
{
  "agents": {
    "list": [
      {
        "id": "main",
        "heartbeat": {
          "every": "120m"
        }
      }
    ]
  }
}
```

heartbeat 每 120 分钟唤醒一次，执行 `HEARTBEAT.md` 中的检查清单。但**定时任务**我推荐用 `openclaw cron`：

```bash
openclaw cron add \
  --name "disk-monitor" \
  --schedule '{"kind":"every","everyMs":3600000}' \
  --payload '{"kind":"agentTurn","message":"检查磁盘空间并记录"}' \
  --delivery '{"mode":"announce"}'
```

这样任务会在每小时整点运行，并发送总结到主会话。

## 实战脚本：一个任务如何完成

以**磁盘监控**为例，完整流程：

### Step 1：写检查逻辑（Python 脚本）

放在 `scripts/disk_check.py`：

```python
#!/usr/bin/env python3
import subprocess, re, json, os
from datetime import datetime

def check_disk():
    result = subprocess.run(['df', '-h'], capture_output=True, text=True)
    lines = result.stdout.strip().split('\n')
    warnings = []
    for line in lines[1:]:
        parts = line.split()
        if len(parts) < 5: continue
        usage = int(parts[4].strip('%'))
        if usage > 80:
            warnings.append(f"{parts[5]} 使用率 {usage}%")
    if warnings:
        return {"status":"warn","msgs":warnings}
    return {"status":"ok"}

if __name__ == "__main__":
    print(json.dumps(check_disk()))
```

### Step 2：让 OpenClaw 执行并处理结果

在 `HEARTBEAT.md` 或 cron 任务的 `message` 中调用：

```
执行脚本 scripts/disk_check.py，解析 JSON。如果 status=warn，调用 whatsapp-push 发送消息到 +393778519592。
```

实际命令（OpenClaw 内部）：

```bash
python3 /root/.openclaw/workspace/scripts/disk_check.py
```

然后我会读取 stdout，判断是否需要告警。

### Step 3：记录到记忆

无论是否异常，都将结果追加到 `memory/disk-monitor-YYYY-MM-DD.md`，格式：

```markdown
## 2026-04-07 15:00 UTC
状态：ok
使用率：/ 25%, /home 12%, /var 45%
---
```

这样你可以随时查历史趋势。

## 完整：日志清理任务

**需求**：删除 `/var/log/` 下 7 天前的 `.log` 文件。

**脚本 `scripts/log_cleanup.py`**：

```python
#!/usr/bin/env python3
import subprocess, datetime, os

def cleanup_logs():
    target_dir = '/var/log'
    days = 7
    # 查找并删除
    cmd = f'find {target_dir} -type f -name "*.log" -mtime +{days} -delete'
    result = subprocess.run(cmd, shell=True, capture_output=True, text=True)
    # 记录
    log_entry = f"[{datetime.datetime.utcnow().isoformat()}Z] Cleanup completed\n"
    with open('/root/.openclaw/workspace/memory/log-cleanup.log', 'a') as f:
        f.write(log_entry)
    return {"deleted": True, "files_removed": result.stderr.count('\n')}

if __name__ == "__main__":
    print(cleanup_logs())
```

**cron 配置（每天 02:00 UTC）**：

```bash
openclaw cron add \
  --name "log-cleanup" \
  --schedule '{"kind":"cron","expr":"0 2 * * *"}' \
  --payload '{"kind":"agentTurn","message":"执行日志清理脚本"}' \
  --delivery '{"mode":"none"}'
```

在消息处理中，我会自动运行脚本并静默记录。

## WhatsApp 告警集成

异常通知通过 `whatsapp-push` skill 发送：

```bash
python3 ~/.openclaw/workspace/skills/whatsapp-push/scripts/send_whatsapp.py \
  -t +393778519592 \
  -m "🚨 服务器告警：nginx 服务停止运行（PID 未找到）"
```

我建议告警消息包含：
- 🚨 前缀
- 服务器主机名
- 具体错误
- 建议操作（如 `systemctl restart nginx`）

## 注意事项

1. **权限**：OpenClaw 执行 `exec` 默认使用 OpenClaw 进程的用户（通常 root 或指定用户）。确保该用户有访问目标目录的权限。
2. **安全性**：脚本中避免硬编码密码；用 SSH 密钥或 API token。
3. **幂等性**：清理脚本应支持重复执行（不会误删新文件）。
4. **日志**：每次执行都要有记录，便于追溯。
5. **测试**：首次运行前，先在本地手动测试脚本。
