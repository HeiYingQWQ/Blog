## 引言
随着技术发展，Hexo + Stellar 博客搭建全流程 已经成为开发者必备技能。
本文将手把手带你从零开始，完成搭建与配置。全程基于最新稳定版本，适合新手跟随。

## 前置条件
在开始之前，请确保你拥有：

- [x] 一台能运行命令行的机器（Linux/macOS/Windows WSL2）
- [x] 已安装 Git（用于版本控制和部署）
- [x] 有一个 GitHub 账号（用于托管代码）
- [x] 基本的命令行操作能力

## 第一步：安装 Hexo
确保已安装 Node.js (>= 12.x)，然后全局安装 Hexo：
```bash
npm install -g hexo-cli
```

## 第二步：初始化博客
选择目录创建博客项目：
```bash
hexo init my-blog
cd my-blog
npm install
```

## 第三步：安装 Stellar 主题
方式一：通过 npm 安装（推荐）
```bash
npm install hexo-theme-stellar
```
然后在 `_config.yml` 中设置 `theme: stellar`。
方式二：克隆主题仓库到 `themes/stellar/`：
```bash
git clone https://github.com/xaoxuu/hexo-theme-stellar themes/stellar
```

## 第四步：基础配置
修改博客根目录 `_config.yml`：
```yaml
title: 我的博客
author: 你的名字
language: zh-CN
theme: stellar
permalink: archives/:title/
```
如需更多主题配置，请编辑 `_config.stellar.yml`。

## 第五步：本地预览
运行以下命令启动本地服务器：
```bash
hexo s
```
浏览器访问 `http://localhost:4000` 即可看到效果。

## 第六步：部署到 GitHub Pages
1. 在 GitHub 创建仓库 `<username>.github.io` 或你喜欢的项目仓库
2. 安装 hexo-deployer-git：
```bash
npm install hexo-deployer-git --save
```
3. 修改 `_config.yml` 添加部署配置：
```yaml
deploy:
  type: git
  repo: https://github.com/你的用户名/你的仓库.git
  branch: gh-pages
```
4. 生成并部署：
```bash
hexo clean && hexo g && hexo d
```
稍等片刻，访问 `https://你的用户名.github.io` 查看效果。
## 常见问题
### 1. hexo 命令找不到或执行失败
确保 Node.js 和 hexo-cli 已正确安装，并检查 npm 全局路径是否在 $PATH 中。

### 2. 部署后页面显示 404
确认 GitHub Pages 配置正确，分支为 `gh-pages`，源目录为根目录。

### 3. 主题配置修改不生效
某些 Stellar 主题配置需要重启 Hexo 服务或清除缓存（`hexo clean`）才能生效。
