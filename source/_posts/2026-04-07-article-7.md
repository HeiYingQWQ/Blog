## 引言

Tailscale 是基于 WireGuard 的现代 VPN 解决方案，它让内网穿透变得极其简单。无需公网 IP、无需配置端口转发，就能安全地访问你的所有设备。

### 为什么选择 Tailscale

- **Zero Trust 安全模型**：每个连接都加密且认证
- **零配置**：自动密钥管理，无需手动配置
- **跨平台**：支持 Linux、Windows、macOS、iOS、Android
- **开源**：核心协议开源，可自托管协调服务器

---

## 前置条件

- 至少一台可访问公网的设备（作为出口节点，可选）
- 需要访问的设备：服务器、笔记本电脑、手机等
- 所有设备安装 Tailscale 客户端

---

## 安装与配置

### 步骤 1：注册 Tailscale 账号

访问 https://tailscale.com 注册（免费套餐足够个人使用）。

获取 Auth Key（便于自动化配置）：
```
Settings → Auth keys → Generate auth key
```

### 步骤 2：安装客户端

**Ubuntu/Debian:**
```bash
curl -fsSL https://tailscale.com/install.sh | sh
systemctl enable --now tailscaled
```

**CentOS/RHEL:**
```bash
yum install tailscale
systemctl enable --now tailscaled
```

**macOS:**
```bash
brew install tailscale
sudo tailscale up
```

**iOS/Android**: 从应用商店下载 Tailscale App，扫描二维码登录。

---

### 步骤 3：连接设备

使用 Auth Key 快速连接：
```bash
tailscale up --authkey tskey-xxxxxx
```

或交互式登录：
```bash
tailscale up
# 浏览器打开 URL 完成登录
```

连接成功后，设备会获得一个 `100.x.y.z` 的 IP 地址，这个 IP 在 Tailscale 网络内可达。

## 访问控制列表（ACL）

Tailscale 默认允许所有设备互访。如需更细粒度控制，配置 ACL：

在控制台 https://login.tailscale.com/admin/acl 编辑 JSON：

```json
{
  "acls": [
    {
      "src": ["autogroup:members"],
      "dst": ["100.64.0.0/10:*"],
      "action": "accept"
    }
  ],
  "groups": {
    "autogroup:admins": ["user@example.com"],
    "autogroup:members": ["autogroup:admins"]
  }
}
```

这条规则允许管理员访问所有端口，其他成员只能访问允许的端口。

## 与 OpenClaw 网关集成

如果你的服务器运行 OpenClaw，可以通过 Tailscale 安全地远程管理。

### 配置步骤

1. **在 OpenClaw 服务器上安装 Tailscale**
   ```bash
   tailscale up --authkey <key>
   ```

2. **获取服务器的 Tailscale IP**
   ```bash
   tailscale ip
   # 输出: 100.101.102.103
   ```

3. **从你的电脑通过 Tailscale 访问**
   ```bash
   # SSH 到服务器
   ssh user@100.101.102.103

   # 访问 OpenClaw API (无需域名)
   curl http://100.101.102.103:8080/api/status
   ```

4. **配置 OpenClaw 的 `gateway.bind`**
   ```yaml
   gateway:
     bind: 0.0.0.0:8080
     remote:
       url: http://100.101.102.103:8080
   ```

   这样，移动设备通过 Tailscale 网络访问网关。

## 子网路由（访问整个内网）

如果你的其他设备（如树莓派、NAS）不在 Tailscale 上，可以用一台设备作为子网路由器：

在担任路由器的设备上执行：
```bash
# 开启 IP 转发
sudo sysctl -w net.ipv4.ip_forward=1

# 宣告子网
tailscale up --advertise-routes=192.168.1.0/24

# 在其他设备上允许该路由
tailscale up --accept-routes=100.101.102.103/32
```

现在你可以通过 Tailscale IP 访问整个 `192.168.1.0/24` 网段。

## 性能与安全考虑

### 性能
- Tailscale 使用 WireGuard，开销极低
- 加密/解密由内核模块处理，性能接近原生
- 支持 UDP hole-punching，P2P 直连优先

### 安全
- 每个连接有加密密钥（每 session 变化）
- 设备必须通过认证才能加入网络
- ACL 提供应用层访问控制
- 支持 Exit Node（统一出口 IP）

## 常见问题

**Q: 设备无法连接？**  
A: 检查 `tailscale status`，确保设备在线。防火墙放行 UDP 端口 41641。

**Q: 无法访问某设备端口？**  
A: 检查 ACL 规则，确认允许该端口。另外检查目标设备的防火墙。

**Q: 如何设置统一出口 IP（Exit Node）？**  
A: 在出口节点上 `tailscale up --advertise-exit-node=true`，其他节点 `tailscale up --exit-node=100.x.y.z`。

**Q: 移动设备不固定 IP？**  
A: 可以在 Tailscale 控制台为设备设置 `--fixed-ip`，或使用 DNS 名（`device-name.your-tailnet.ts.net`）。
